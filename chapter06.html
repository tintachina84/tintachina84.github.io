Chapter 6 -인증 및 권한 부여를 사용하여 REST 엔드포인트 보호하기

이제 Spring Security를 사용하여 이러한 REST 엔드포인트를 안전하게 보호하는 방법을 배우게 될 것입니다. REST 엔드포인트에 대한 토큰 기반 인증 및 권한 부여를 구현할 것입니다. 성공적인 인증은 두 가지 유형의 토큰을 제공합니다 - 액세스 토큰으로서의 JavaScript Object Notation (JSON) 웹 토큰 (JWT) 및 응답으로 제공되는 리프레시 토큰. 이 JWT 기반 액세스 토큰은 보호된 URLs에 액세스하는 데 사용됩니다. 리프레시 토큰은 기존 JWT가 만료된 경우 새 JWT를 요청하는 데 사용되며 유효한 요청 토큰은 사용할 새 JWT를 제공합니다.

사용자를 ADMIN 및 USER와 같은 ROLE과 연결할 것입니다. 이러한 ROLE은 REST 엔드포인트에 액세스할 수 있는지 확인하기 위한 권한으로 사용될 것입니다. 또한 간단히 교차 사이트 요청 위조(CSRF) 및 교차 출처 리소스 공유(CORS)에 대해 이야기할 것입니다.

이 장에서 다룰 섹션은 다음과 같습니다.

  • Spring Security와 JWT를 사용한 인증 구현

  • JWT를 사용하여 REST API 보안 강화하기

  • CORS 및 CSRF 설정

  • 인가(Authorization)의 이해

  • Security 테스트

6.1 Spring Security와 JWT를 사용한 인증 구현

Spring Security는 보일러플레이트 코드를 작성할 필요 없이 기업 애플리케이션 보안을 구현할 수 있게 해주는 라이브러리 모음으로 이루어진 프레임워크입니다. 이 장에서는 Spring Security 프레임워크를 사용하여 토큰 기반(JWT) 인증 및 권한 부여를 구현할 것입니다.

Spring Security는 인증을 위해 토큰을 지원합니다.

토큰은 문자열입니다.

5rm1tc1obfshrm2354lu9dlt5reqm1ddjchqh81 7rbk37q95b768bib0j f44df6suk1638sf78cef7 hfolg4ap3bkighbnk7inr68ke780744fpej0gtd 9qflm999o8q

우리는 이러한 문자열을 인가(Authorization) 흐름을 통해 stateless한 HTTP 엔드포인트 혹은 리소스에 접근하도록 사용할 것입니다.

2장에서 DispatcherServlet에 대해 배웠습니다. DispatcherServlet은 클라이언트 요청과 REST 컨트롤러 사이의 인터페이스입니다. 따라서 토큰 기반 인증 및 권한 부여를 위한 로직을 넣고 싶다면, 요청이 DispatcherServlet에 도달하기 전에 이 작업을 수행해야 합니다. Spring Security 라이브러리는 서블릿에 사전 필터(pre-filter)를 제공하여 요청이 DispatcherServlet에 도달하기 전에 처리됩니다. 사전 필터는 실제 서블릿에 도달하기 전에 처리되는 서블릿 필터입니다. Spring Security의 경우 이 서블릿은 DispatcherServlet입니다.
